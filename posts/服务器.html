GRUB（GNU GRand Unified Bootloader） 广泛应用于 Linux 系统中，是启动管理的重要工具。它在 Linux 系统的安装过程中作为默认引导加载程序，在计算机启动时运行，负责加载和启动操作系统内核。

  • 引导操作系统：GRUB 在计算机开机后为用户提供选择菜单，选择要启动的操作系统，比如 Linux 或 Windows。
  • 多重引导支持：可以同时管理多个操作系统，方便用户在不同系统间切换。
  • 配置灵活：用户可以通过配置文件自定义启动菜单和启动参数。

GRUB 需要安装在硬盘的主引导扇区上，确保系统能正确引导。

Note：

  • 计算机开机 → BIOS/UEFI → GRUB 加载器 → 选择操作系统 → 启动操作系统
  • 主引导扇区，MBR（Master Boot Record，主引导记录），位于硬盘的第一个扇区，即扇区 0。

3. 性能测试

YABS (Yet Another Benchmarking Script) ，实用脚本，用于测试服务器或虚拟专用服务器（VPS）的性能。

它会运行一系列的基准测试，来评估你的系统在以下几个方面的表现：

  • CPU 性能：你的处理器有多快。
  • 磁盘 I/O 性能：你的存储设备读写数据的速度有多快。
  • 网络速度：你的互联网连接有多快。

命令简述

1.yabs

2.跳过网络，测GB5

3.跳过网络和磁盘，测GB5

4.改测GB5不测GB6

说明

  • 1 是运行 yabs 脚本的最基本命令，它会进行完整的基准测试，包括网络、磁盘和 CPU 测试。
  • 2、3、4 都是有选择地执行 yabs 脚本。
    • 2 跳过网络措施，并使用 Geekbench 5 (GB5) 进行 CPU 性能测试。
    • 3 同时跳过网络和磁盘测试，并使用 GB5 进行 CPU 基准测试。
    • 4 执行所有测试（网络、磁盘、CPU），但确保 CPU 测试使用 GB5。

命令详解

  • curl：命令行下载工具，用于通过 URL 传输数据。简之，从互联网上下载数据。

  • -s：”silent”，告诉 curl 采用静默模式。它会抑制进度条和错误消息，让输出更整洁。

  • -L：告诉 curl 遵循重定向。如果 yabs.sh 脚本被移动到新的 URL，curl 会跟随重定向找到它。

  • yabs.sh：这是 YABS 脚本本身的 URL。curl 会下载这个脚本。

  • |：这是一个“管道”符号。它把左边命令（下载的脚本）的输出，作为右边命令的输入。

  • bash：这是一个命令解释器（一个“Shell”）。它会接收 curl 下载的脚本并执行它。

  • bash -s：告诉 bash 从标准输入读取命令（也就是从 curl 管道过来的脚本）。

  • --：这是一个标准约定，表示 bash 自身选项的结束，之后的所有参数都将传递给正在运行的脚本（这里是 yabs.sh）。

-if5 是 YABS 脚本特有的选项：

  • -i：”ignore network”，告诉 YABS 不要执行网络速度测试。

  • -f：”ignore fio”（fio 是 YABS 用于磁盘 I/O 测试的工具），告诉 YABS 不要执行磁盘速度测试。

  • -5：告诉 YABS 使用 Geekbench 5 (GB5) 进行 CPU 基准测试，而不是默认或最新版本（通常是 Geekbench 6）。

4. 流媒体及IP质量测试

这些命令工具主要用来检测你的服务器或 VPS 是否能访问特定的流媒体服务（比如 Netflix、Disney+ 等），以及你的 IP 地址的质量。

  • 流媒体解锁检测：检查你的服务器的 IP 地址是否能成功解锁并观看特定地区的流媒体内容。很多流媒体服务会限制内容的播放区域（地理限制），如果你用的 IP 不在允许的区域内，或者被识别为数据中心 IP 而被屏蔽，就无法观看。
  • IP 质量检测：评估你的 IP 地址的“干净”程度，例如是否被识别为数据中心 IP、是否被很多服务商屏蔽、是否在全球范围内能正常访问。

1.最常用版本

2.原生检测脚本（倾向）

3.准确度最高

4.IP质量体检脚本（倾向）

5.一键修改解锁DNS

简说

  • 1、2、3 都是流媒体解锁检测脚本，但检测侧重点有所不同，比如检测的平台种类或准确性
  • 4 比较全面，包括鸡场信息、IP类型熟悉、风险评估、风险因子、流媒体及AI服务解锁检测、邮局连通性及黑名单检测。
  • 5 通过修改你系统的 DNS 设置，尝试通过 DNS 解锁的方式来绕过流媒体的地理限制，以解锁流媒体。（请注意，运行这类修改系统配置的脚本需要谨慎，最好在了解其具体功能和风险后再操作。）

命令详解

  • curl：下载数据。
  • check.unlock.media：短网址，重定向服务，指向一个流媒体解锁检测脚本（原始长网址）。
  • -sL：静默下载（输出干净），遵循重定向（确保下载到最终的脚本文件）。
  • bash <(...)：“进程替换”（Process Substitution），一个常用技巧。curl ... 命令的输出，不会保存到文件中，而是直接作为标准输入传递给 bash 命令，由 bash 执行接收到的脚本。（简之，进程替换，直接执行脚本）
  • IP.Check.Place：短网址重定向服务，指向一个 IP 质量检测脚本。
  • wget：命令行下载工具，侧重于下载数据并保存到本地。curl 的功能与之类似，但更侧重于传输数据。wget 会将该脚本下载到当前目录，并命名为 dns-unlock.sh。
  • &&：逻辑与操作符。它表示前一个命令成功执行后，才执行后一个命令。
  • bash dns-unlock.sh：执行刚刚下载到本地的 DNS 修改脚本。

网络文件的长网址与短网址

长网址（Full URL）是一个直接指向文件在互联网上位置的 URL。它包含了协议、域名、以及文件在服务器上的完整路径和文件名。例如， https://github.com/user/repo/raw/main/script.sh 。长网址稳定、直接，将长网址粘贴到浏览器或下载工具里，通常就能直接下载到文件。

短网址（Short URL）是一个简短易记的 URL。例如， check.unlock.media。短网址本身并不直接指向最终的文件，而是一个重定向服务的入口（该服务由短网址服务商提供）。当短网址被访问时，短网址服务会返回真正完整的地址（长网址）。短网址方便、简洁，其背后隐藏着一个重定向过程。

用户将文件的长网址输入给短网址服务平台，平台会生成一个短网址。如果文件的真实地址发生变化，短网址的维护者只需更新短网址服务中的重定向目标，而无需更改已经发布的短网址。

  • 短网址简短，易于记忆、输入和分享。
  • 短网址服务可以隐藏真实的源地址，也可以做统计追踪（通常提供点击量等统计数据）。

发生请求到短网址时，短网址的服务器通常会返回一个 3xx 系列的重定向状态码（而不是直接返回文件内容），并在响应头中用 Location 字段携带真实地址。

以 curl -L -s check.unlock.media 为例，该命令中 curl 使用 -L 选项，它就会读取 Location 字段，然后自动向新的真实地址再次发送请求。这样，curl 最终就能从真实地址下载到脚本的内容。（如果 curl 命令中没有 -L 选项，那么它接收到重定向响应时，只会显示重定向信息，而不是自动追随新地址，也就不会去下载实际的脚本内容。）

curl 与 wget

curl 和 wget 都是 Linux/Unix 系统下非常常用的命令行工具，用于从网络上下载文件或与服务器进行数据交互。它们有很多相似之处，但也存在关键的区别，这决定了你在不同场景下选择哪一个。

curl 与 wget 的核心区别

特性/功能 curl wget
主要用途 数据传输工具。设计用于与各种协议进行数据交互（上传和下载），更像一个“瑞士军刀”。 网络下载器。主要设计用于从 Web 下载文件和镜像网站，更专注于下载。
默认输出 默认将内容输出到标准输出 (STDOUT)，即直接显示在终端上。若要保存到文件，需要使用 -o 或 -O 选项。 默认将内容保存到本地文件，文件名为 URL 的最后部分。
协议支持 支持非常广泛的协议，包括 HTTP, HTTPS, FTP, FTPS, SCP, SFTP, TFTP, DICT, FILE, GOPHER, IMAP, POP3, SMTP, RTSP, RTMP, SMB, SOCKS 等。 主要支持 HTTP, HTTPS, FTP。
递归下载 不内置支持递归下载。你需要手动编写脚本或结合其他工具来实现。 强大的递归下载能力，可以下载整个网站、目录结构，并遵循链接，非常适合网站镜像或离线浏览。
文件上传 支持文件上传（例如使用 -T 选项）。 不直接支持文件上传，通常需要配合其他工具（如 wput）。
库支持 基于 libcurl 库构建，这是一个可供其他程序使用的跨平台库，提供了很大的灵活性。 仅限命令行工具，没有对应的库。
错误恢复 支持通过 —retry 和 —retry-delay 选项重试，但不直接支持中断续传（需要手动指定断点 -C -）。 内置强大的断点续传功能（使用 -c 选项），在网络中断后能从上次停止的地方继续下载。
后台运行 通常不内置后台运行功能（但可以通过 & 或 nohup 实现）。 默认支持后台下载，即使终端关闭也能继续下载（使用 -b 选项）。
复杂请求 对 HTTP 请求的控制更精细，例如可以自定义请求头、发送 POST/PUT 数据、处理 cookies 等，非常适合与 API 交互和调试。 对 HTTP 请求的控制相对有限，主要用于简单的文件下载。
便捷性 对于简单的下载，需要额外指定保存文件名的选项。 对于简单的文件下载，直接指定 URL 即可，更简洁。

何时选择 curl？

  • 当你需要与 Web API 进行交互（发送 POST/PUT 请求、自定义请求头、处理 JSON/XML 数据等）。
  • 当你需要测试服务器的响应，并直接在终端查看内容（因为默认输出到 STDOUT）。
  • 当你需要支持更广泛的协议类型进行数据传输。
  • 当你需要进行文件上传。
  • 当你希望在脚本中对网络请求有更细粒度的控制。

何时选择 wget？

  • 当你需要简单地下载一个或多个文件到本地。
  • 当你需要下载整个网站或目录以进行离线浏览或备份（递归下载）。
  • 当你可能遇到网络不稳定，需要自动断点续传大文件。
  • 当你希望下载任务在后台运行。
  • 当你需要一个更**“傻瓜式”**的下载工具，即只关心“获取文件”这个结果。

总结

可以这样概括：

  • curl 更像是一个多功能的“网络瑞士军刀”，它能做的事情非常多，不仅限于下载，还包括上传、API 交互、协议测试等，但对于简单的下载，需要额外指定保存文件的选项。
  • wget 则是一个专注于“下载”的强大工具，尤其擅长处理大文件、断点续传和递归下载，使用起来相对更简单直接。

在日常使用中，很多简单的下载任务两者都能胜任，但深入到特定需求时，它们的优势便会体现出来。
