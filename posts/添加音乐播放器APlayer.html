PJAX 是一种结合了 AJAX 和 HTML5 的 History API（pushState）的网页开发组合技术，用于实现网页局部刷新和无刷新加载内容的效果，同时更新浏览器地址的 URL。用户点击链接或进行导航时，只更新页面的一部分内容，而无需重新加载整个页面。

  • 通过 AJAX 请求实现网页局部刷新。只请求替换内容，减少服务器负载。只更新替换内容，避免完整刷新带来的闪烁和等待时间，页面切换更加流畅，提升用户体验。
  • 通过 pushState，可以在不刷新页面的情况下，改变浏览器地址栏的 URL，保持浏览器历史记录。用户可以使用后退、前进按钮正常导航。

典型的工作流程：

  1. 用户点击链接。
  2. PJAX 拦截该点击事件，通过 AJAX 请求对应的内容。
  3. 获取新内容后，用 JavaScript 动态替换页面的部分区域（如内容区域）。
  4. 使用 pushState 更新浏览器地址栏的 URL，保持导航的一致性。

your work in html

  1. 引入 PJAX 库
  1. 发起 PJAX 请求
  • selector：指定触发 PJAX 请求的 click 事件绑定的元素。CSS 选择器或 jQuery 对象。
  • [container]：PJAX 容器，指定加载内容的目标 DOM 元素。CSS 选择器。
  • options 选项：
    1. fragment：从 ajax 响应中提取的内容。CSS 选择器。
    2. timeout：ajax 超时时间（以毫秒为单位），超时则强制进行完全刷新。
    3. push：默认 true，使用 window.history.pushState 改变地址栏 url（ 更新历史记录 ）。
    4. replace：默认 false，使用 window.history.replaceState 改变地址栏 url（不添加历史记录 ）。
    5. ……

（一般地，PJAX 请求的新页面与当前页面布局一致，PJAX 容器与 fragment 一致。）

示例：

  1. PJAX 请求的善后处理

正常获取新页面内容都是完全刷新页面，执行页面 scripts 以集成功能模块。PJAX 请求只负责替换新页面的内容到当前页面，而不会自动加载替换内容中的 <script> 。此外，当前页面中原有的 <script> 也并不会被再次执行。这可能会导致页面部分功能缺失。因此，你需要在 PJAX 请求后重新加载必要的 scripts（烦躁！），推荐使用 pjax:end 事件。

<link> 同理？

（慎重使用 AJAX！页面小崩坏是常有的事！但知道 AJAX 还不用的，也是神人了！）

首先，请不要在【提取的内容】和【PJAX 容器】中内嵌 <script>（简单粗暴！如果内嵌就按自己想法操作吧……）然后，查看当前页面原有的 <script>（网页 F12 元素）。最后，将【必要的 JS 程序】添加到 pajx_reload 函数中，在 pjax:end 事件发生时调用。

<script> 为内联代码，或者是引入外部 JS 文件。在发起 PJAX 请求后，原页面在 <script> 中定义的全局函数和变量通常还能访问到，只要没有在内容替换过程中被删除或重置。页面集成的功能模块一般以函数的形式存在，函数执行时，功能模块生效。这些函数的执行方式是我们的焦点，我将其简单地划分为调用和事件驱动。

  1. 如果功能模块函数仅是调用（手动调用，or 使用 call()/apply() 等），或者绑定了 DOMContentLoaded、load 之类的事件，那么这些函数只会执行一次。PJAX 请求后，这些功能模块就会失效。我们的工作就是将这些函数添加到 pajx_reload 函数中。（necessary scripts!）
  2. 如果功能模块函数绑定 click、scroll 之类的事件，这些事件可以重复触发，触发时功能模块即生效。（no need!）

如果必要的 scripts 是整个内联代码（是人类吗？），重新插入该 <script> 到 DOM 中。 设置该 <script> 带有 date-pjax 属性，方便 JS 获取。或者，使用 <div class="pjax-reload"></div> 包裹该 <script>。（自调用函数同理。）

Tip：将 <script> 标签重新插入到 DOM 后，浏览器会自动解析并执行。

  1. 设置 pjax:end 事件要触发的必要 scripts

网页 F12 元素，查看原页面必要的脚本，添加到函数 pjax_reload 中。

  1. 如果必要 scripts 是函数，在函数 pjax_reload 中执行这些必要函数。
  2. 如果必要 scripts 是整个 <script>，为这些 <script> 添加属性 data-pjax（修改 Fluid 主题源码），以便于获取。在函数 pjax_reload 中获取这些 <script> 并添加到 DOM，触发 HTML 解析执行。

Fluid 主题的布局模板文件中采用两种写法注入 <script> 脚本（具体内容请自行看 Fluid 主题文件）。

  1. 自定义函数 import_script()，参数内容为 <script> 脚本。直接一个 date-pjax 属性的加！
  2. 自定义函数 js_ex()，传入参数，参数作为 <script> 的 src 属性值以及其他属性。

这里给出三个例子（其他自行解决）。

示例 1：<script src="/js/boot.js"></script>

  1. 函数 Fluid.boot.registerEvents(); 在 HTML 解析完成（DOM 树建立完毕）后执行。PJAX 请求后，该函数失效。
    • 函数 Fluid.events.billboard(); 只是在控制台输出 Fluid 字符，属于非必要 script。
    • 函数 Fluid.events.registerNavbarEvent(); 是导航栏特效（鼠标向下滚动一定距离，导航栏采用深色样式），属于必要 script。
    • ……
  2. 函数 Fluid.boot.refresh 在生成 HTML 时调用，它与页面功能模块生效无关。

直接在函数 pjax_reload 中调用这些必要 scripts。

示例 2：<script src="/js/img-lazyload.js"></script>

引用的外部 JS 文件——img-lazyload.js，整个 JS 程序是自调用函数，用于显示懒加载图片，属于必要 script。

因为该必要 script 是自调用函数，是 <script> 的外部引用脚本。

先直接给 <script> 标签添加 data-pjax 属性，方便获取。layout\_partials\scripts.ejs 中使用 scripts\helpers\url.js 中的函数 js_ex 注入 <script>，刚好该函数多设计了一个 ex 参数，直接修改代码传参 date-pjax——js_ex(theme.static_prefix.internal_js, 'img-lazyload.js', 'data-pjax')。

然后在函数 pjax_reload 中获取它，并将它重新添加到 DOM，触发 HTML 解析执行。

示例 3：<script src="https://lib.baomitu.com/tocbot/4.20.1/tocbot.min.js" type="text/javascript" charset="UTF-8"></script>

引用的外部 JS 文件——tocbot.min.js，整个 JS 程序是自调用函数，用于生成文章目录，属于必要 script。

因为该必要 script 是自调用函数，是 <script> 的外部引用脚本。（处理思路和示例 2 一样，但略复杂）

函数 import_script 将【<script> 参数】注入到生成的 HTML。 <script> 参数内部执行函数 Fluid.utils.createScript，通过创建 script 元素的方式引入tocbot.min.js 必要 script；回调函数（绑定 load 事件）在页面所有资源（图片、样式表、脚本等）都加载完后执行。

只要重载一次【<script> 参数】即可使 tocbot 生效，为【<script> 参数】添加 data-pjax 属性，然后函数 pjax_reload 做后续处理（示例 2 已经处理）。（执行时机与示例 2 不同，示例 3 是为了触发 HTML 解析）。

因为是通过新建 script 元素的方式引入 tocbot.min.js 必要 script，如果不断发起 PJAX 请求，<script src="tocbot.min.js"> 元素会不断堆积。可以简单地修改一下函数 createScript，防止元素堆积。

声明函数 createScriptST，内容 copy 函数 createScript，新增参数来设置 script 属性，以方便获取。（暂时新建函数，以免其他调用函数 createScript 的地方出问题）

【<script> 参数】部分改为调用函数 createScriptST，多传入一个参数 'tocbot'。在调用此函数前，删除页面已有的<script func="tocbot" src="tocbot.min.js">，防止 PJAX 请求导致该元素堆积。

其他的功能模块，自行处理：

  • code-widget，代码窗口小部件，使用 clipboard.js 粘贴复制插件，fluid\layout\_partials\plugins\code-widget.ejs
  • anchor，小标题锚点，fluid\layout\_partials\plugins\anchorjs.ejs
  • /js/local-search.js，搜索功能，fluid\layout\_partials\header\navigation.ejs
  • /js/color-scheme.js，白天黑夜模式切换，fluid\layout\_partials\head.ejs
  • ……
